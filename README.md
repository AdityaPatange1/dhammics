# Dhammics

> A minimal, fast, static blogging platform for essays on the Dhamma.

Dhammics is built with vanilla HTML, CSS, and JavaScript — **no bundler, no
framework, no tracking** — so the site stays fast, transparent, and easy to
host anywhere. Essays are authored as plain Markdown files in
`content/posts/`, and the site ships as plain files ready for GitHub Pages.

- **Frontend-first** — every page is a real HTML document you can open from disk.
- **No build step required** for development. `npm run dev` serves the repo root.
- **Light green + white palette** with proper dark mode (system-aware, overridable).
- **Aesthetic motion** — scroll-reveals, subtle hero gradients, polite hovers — with
  `prefers-reduced-motion` respected.
- **Lucide icons** via a single CDN script tag.
- **Admin panel** for composing new posts in Markdown, with live preview and a
  draft-export flow that produces the exact file + manifest entry needed to
  publish via a commit.

## Getting started

Requires **Node.js 18+** and **npm**.

```bash
git clone <repository>
cd dhammics
npm install
npm run dev          # http://localhost:4173
```

The dev server is just a static file server — no bundling happens.

## Scripts

| Script            | What it does                                                  |
| ----------------- | ------------------------------------------------------------- |
| `npm run dev`     | Starts a local static server on port `4173`.                  |
| `npm run build`   | Regenerates the manifest, emits `dist/` + sitemap + RSS.      |
| `npm run validate`| Validates the manifest and every post's front-matter.         |
| `npm run lint`    | Runs ESLint, Stylelint, and HTMLHint.                         |
| `npm run format`  | Prettier-format every source file.                            |
| `npm run check`   | `validate → lint → build` (used by CI).                       |

## Authoring a new post

### Option A — via the admin panel

1. `npm run dev`, open <http://localhost:4173/login.html>.
2. Sign in with the default credentials **`admin`** / **`dhamma123`**.
   (These are client-side only — see the security note below.)
3. Write the post in the composer. Live preview is one tab away.
4. **Save draft** — stays in your browser and renders on the homepage with a
   "Draft (local)" flag for private preview.
5. **Export & publish** downloads two files:
   - `your-slug.md` → commit to `content/posts/`
   - `your-slug.manifest.json` → merge this entry into
     `content/posts/manifest.json` (or simply re-run `npm run build`, which
     regenerates the manifest from the markdown files).
6. Commit, push, and GitHub Pages will redeploy automatically.

### Option B — directly in Markdown

Create `content/posts/my-new-post.md` with YAML front-matter:

```md
---
title: "The Art of Sitting Still"
slug: the-art-of-sitting-still
date: 2026-05-01
author: "Dhammics"
description: "A short essay on the posture that makes the path possible."
cover: https://images.unsplash.com/photo-.... (pre-hosted)
tags: ["Meditation", "Practice"]
readingTime: 5
---

# The Art of Sitting Still

The body quiets first...
```

Then:

```bash
npm run build         # regenerates manifest.json
npm run validate      # sanity-checks
git add . && git commit -m "post: the-art-of-sitting-still" && git push
```

Images should be **pre-hosted** on a CDN (Unsplash, your own bucket, GitHub
user-content, etc.) and referenced via standard Markdown image syntax.

## Theming

- The default theme follows `prefers-color-scheme`.
- Users can override it with the toggle in the header; the preference is
  persisted in `localStorage` under `dhammics:theme`.
- Colors, spacing, and motion are all tokens in
  [`assets/css/variables.css`](assets/css/variables.css). Change them there.

## Authentication note

The admin panel is protected by a purely client-side check — **this is by
design** for a statically-hosted GitHub Pages site, where no server-side
runtime is available. It is enough to gate the UI from casual visitors.

**Do not** rely on this for sensitive operations. For production use:

- Change the default credentials by setting `dhammics:creds` in `localStorage`:
  ```js
  localStorage.setItem('dhammics:creds', JSON.stringify({
    username: 'your-admin',
    password: 'a-longer-password'
  }));
  ```
- Or replace `assets/js/auth.js` with a call to your own auth service (Clerk,
  Auth0, Firebase, etc.).

## Deployment

A ready-to-use workflow lives at [`.github/workflows/deploy.yml`](.github/workflows/deploy.yml).
Enable **GitHub Pages → Source: GitHub Actions** on the repo, push to `main`,
and the workflow will:

1. Install dependencies.
2. Run `npm run check` (validate → lint → build).
3. Upload `dist/` as the Pages artifact.
4. Deploy.

For any other host, run `npm run build` and serve the resulting `dist/`
folder as static files.

## Accessibility

- Skip-to-content link on every page.
- All interactive controls have visible focus rings.
- `prefers-reduced-motion` disables non-essential animation.
- Icons expose `aria-label` on meaningful buttons.
- Color contrast passes WCAG AA in both themes.

## Performance

- Zero runtime dependencies; the only third-party scripts are Lucide (icons)
  and Google Fonts, both loaded with `defer` / `preconnect`.
- Pages are under ~25 KB of HTML+CSS+JS each (uncompressed).
- Images use `loading="lazy"` and `decoding="async"`.
- `IntersectionObserver` is used for scroll-reveals with delay-based stagger.
